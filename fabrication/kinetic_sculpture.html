Kinetic Sculpture

I built a small kinetic sculpture where a sign moves up and down as a motor rotates. I later made a second version with a motor driver and potentiometer so the speed of the motion could be controlled.

Project overview

The goal was to create a simple moving sculpture using a mechanical linkage. I wanted the sculpture to have a part that moved up and down through the box as the motor rotated.

During testing, I realized that the moving piece needed to stay visible and not slip too far into the box. I switched to a sign because it could move up and down more reliably while still keeping the motion easy to see.

Designing the movement

I experimented with different circle radii to make the motion smoother. The base also needed to be wide enough to roll off the rotating circle properly. Small changes in the size and placement of the moving parts made a big difference in whether the sculpture moved cleanly or jammed.

Once the basic motion worked by hand, I built a motor mount so the motor would stay in place while driving the movement.

Adding motor control

In the next version, I added an L298N motor driver and a potentiometer to control the speed of the motor. Turning the potentiometer changed the voltage input read by the microcontroller, which then adjusted the PWM signal sent to the motor driver.

This made the sculpture more interactive because the motion was no longer just on or off. I could tune how fast the sign moved by changing the potentiometer value.

Code

The code reads the potentiometer value, maps it to a motor speed, and writes that speed using PWM. 

const int A1A = 12;
const int A2A = 14;
const int potPin = 32;

void setup() {
  pinMode(A1A, OUTPUT);
  pinMode(A2A, OUTPUT);
  pinMode(potPin, INPUT);

  ledcAttach(A1A, 5000, 8);
  Serial.begin(115200);
}

void loop() {
  int pot_value = analogRead(potPin);
  int motor_speed = map(pot_value, 0, 4095, 255, 0);

  Serial.println(motor_speed);

  ledcWrite(A1A, motor_speed);
  digitalWrite(A2A, LOW);

  delay(1);
}

Final demo

The final version combined the physical moving sign with motor-speed control. It was a useful way to understand how mechanical motion, electronics, and code all affect each other.

Reflection

This project taught me that even simple motion can require a lot of iteration. The sculpture only worked once the mechanical parts, the motor placement, the circuit, and the code were all aligned with each other.
